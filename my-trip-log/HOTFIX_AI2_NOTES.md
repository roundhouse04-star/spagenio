# 🩹 Hotfix AI 일정 v2 — 기존 여행 모드 검증 + 선택 표시 강화

## 추가 수정 (이전 aifix.zip + 더)

### 문제: 기존 여행 모드인데 새 여행용 입력란을 검증함

**원인**: 사용자가 "기존 여행" 탭으로 전환했지만 **여행 카드를 직접 클릭하지 않으면** `existingTripId`가 null인 상태. getTargetInfo()가 새 여행 입력란(빈 값) 검증으로 fallback → "여행 정보를 입력해주세요" 알림 발생.

### 수정 사항

1. **자동 여행 선택**: 기존 여행 모드 진입 시 가장 최근 여행 자동 선택
2. **모드별 분리된 검증**: 기존 모드는 trip 객체 검증, 새 모드는 입력란 검증
3. **자세한 에러 메시지**: 어떤 필드가 비었는지 명확히 표시
4. **선택 표시 강화**: 선택된 여행 카드를 진한 배경 + 흰 텍스트로 명확히 표시
5. **여행 정렬**: 최신순 (수정일/생성일)

### Before vs After

**Before:**
```
사용자가 "기존 여행" 탭 클릭
→ 여행 목록 보임 (선택 안 함)
→ ChatGPT 버튼 클릭
→ "여행 정보를 모두 입력해주세요" 알림 (혼란!)
```

**After:**
```
사용자가 "기존 여행" 탭 클릭
→ 첫 번째 여행 자동 선택 (강조 표시)
→ 사용자가 다른 여행 클릭하면 즉시 시각적 변경
→ ChatGPT 버튼 클릭
→ 정상 동작
```

## 테스트
1. 홈 → AI 일정
2. "📂 기존 여행" 탭 → ✅ 첫 번째 여행 자동 선택 (진한 배경)
3. 다른 여행 클릭 → ✅ 색상 즉시 변경
4. ChatGPT 클릭 → ✅ Alert 뜸 (즉시 외부 앱 X)
5. "ChatGPT 열기" → ChatGPT 열림
6. 답변 받아서 붙여넣기 → 일정으로 불러오기
7. 성공 alert에 "확인" + "일정 보러 가기" 둘 다 있음
