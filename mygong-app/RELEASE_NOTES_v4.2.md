# v4.2 — 방송·드라마·영화 출연 이력 확장 + 팔로우 자동 싱크

## 한 줄 요약
Wikipedia 파서가 콘서트뿐 아니라 **방송·드라마·영화·뮤직비디오·광고·앨범·수상** 섹션까지 긁어옴. 팔로우 버튼 누르면 자동으로 백그라운드 싱크 돌면서 즉시 이력이 채워짐.

## 해결한 문제

### 문제 1: "아티스트 0명 동기화" 버그

**증상**: 아티스트를 검색해서 등록했는데, 설정에서 "전체 동기화" 눌러도 `아티스트 0명 · 신규 이벤트 0건` 으로 나옴.

**원인**: `syncAllArtists()` 는 `is_following = 1` 인 아티스트만 동기화함. 검색 → 등록만 하고 **팔로우 버튼을 안 누르면** `isFollowing = false` 라 싱크 대상에 안 잡힘.

**해결**: 팔로우 버튼을 누르면 자동으로 `syncOneArtist()` 호출 → 즉시 KOPIS + Wikipedia 데이터 수집.

### 문제 2: 공연 정보만 나오고 출연 이력이 안 나옴

**증상**: Wikipedia 에 가수의 드라마 출연, 예능 출연, 광고, 수상 등이 적혀있는데 앱에서는 콘서트만 보임.

**원인**: 기존 `PERF_SECTION_RE` 정규식이 콘서트·투어·공연 관련 키워드만 매칭.

**해결**: 카테고리별 섹션 매처(`SECTION_CATEGORY_MAP`) 도입. 12개 카테고리로 분류:

| 카테고리 | 아이콘 | 매칭 키워드 |
|---------|-------|------------|
| 콘서트 | 🎤 | 콘서트, 투어, 라이브, 음악회, 리사이틀, 뮤직, 공연, 월드투어 |
| 뮤지컬 | 🎭 | 뮤지컬 |
| 연극 | 🎪 | 연극 |
| 팬미팅 | 💖 | 팬미팅, 팬콘, 쇼케이스 |
| 페스티벌 | 🎉 | 페스티벌 |
| **방송** | **📺** | **예능, 버라이어티, 방송, TV쇼, 라디오, MC, 출연 작품** |
| **드라마** | **🎬** | **드라마, 연속극, 시리즈물** |
| **영화** | **🎥** | **영화, 필모그래피** |
| **뮤직비디오** | **🎞️** | **뮤직비디오, 뮤비, MV** |
| **광고** | **📢** | **광고, CF, 브랜드 모델** |
| **앨범** | **💿** | **앨범, 음반, 디스코그래피, 정규, 미니앨범, 싱글** |
| **수상** | **🏆** | **수상, 시상, 수훈, 영예** |

## 동작 예시

"아이유" 팔로우 시 수집되는 섹션 (Wikipedia):

```
[wiki-events] sections matched:
  "콘서트 및 투어→콘서트",
  "음반→앨범",
  "드라마→드라마",
  "영화→영화",
  "예능 프로그램→방송",
  "뮤직비디오→뮤직비디오",
  "광고→광고",
  "수상 및 후보→수상"
```

각 섹션의 표(wikitable) 와 글머리(`*`) 항목들이 추출되어 Event 로 변환됨.

## 파일 변경

**수정**
- `src/services/providers/wikipediaProvider.ts`
  - `PERF_SECTION_RE` 단일 정규식 → `SECTION_CATEGORY_MAP` 배열 (12개)
  - `categoryForHeading()` 헬퍼 추가
  - `extractPerformanceSections()` 이 `{body, category, heading}[]` 반환
  - `parseSection()` 이 카테고리를 받아서 모든 이벤트에 stamp
- `src/db/schema.ts`
  - `CATEGORIES` 에 방송/드라마/영화/뮤직비디오/광고/앨범/수상/팬미팅 추가
- `app/artist/[id].tsx`
  - `toggleFollow()` 가 "새로 팔로우 시작"일 때 `syncOneArtist()` 를 백그라운드로 호출

**변경 없음** (설계상 유지)
- `syncManager.ts`, `parseData.ts`, `kopisProvider.ts`
- 기존 KOPIS 흐름 그대로 동작 (공연만, 방송 X — KOPIS 는 공연 DB 라서)

## 카테고리 색상 — CSS (선택 사항)

`theme.ts` 에 새 카테고리 색상 키(`catBroadcast`, `catDrama` 등) 가 아직 없으면 UI 에서 기본색으로 떨어짐. 필요 시 theme 에 추가하면 각 카테고리별 배지 색상 분리 가능.

## 알려진 한계

1. **Wikipedia 본문 구조 의존** — 편집자마다 표 형식/섹션 제목이 다르므로 누락 가능. "주요 출연" 같은 애매한 제목은 무시됨.
2. **KOPIS 는 공연만 반환** — 방송·드라마 데이터 없음. Wikipedia 쪽 확장으로만 커버됨.
3. **날짜 없는 항목** — Wikipedia 에 "2023년" 처럼 연도만 있는 항목은 `date = 2023-01-01` 로 저장됨 (정렬·알림 기준 일관성 확보용).
